# Vibe Coding 评分标准

## 使用场景
当需要了解不同题目类型的评估方法时使用此技能。

## 各题目类型评估方法

### ALGORITHM_MODELING（算法与建模）
- 尝试在运行环境中执行候选人的脚本/代码，验证输出正确性
- 检查算法逻辑和边界条件处理
- 分析时间/空间复杂度是否合理

### FEATURE_DEV（功能开发）
- 尝试启动项目，验证核心功能链路是否跑通
- 检查前后端接口对接是否正确
- 审查代码结构和工程规范

### DEBUG_FIX（调试修复）
- 在运行环境中复现问题，确认 bug 是否修复
- 编写/运行测试用例验证修复效果
- 检查是否引入新的回归问题

### DATA_PROCESSING（数据处理与分析）
- 检查数据处理管道的输出结果
- 验证统计计算和分析结论的正确性
- 查看可视化图表的质量

### AGENT_DEV（智能体开发）
- 测试 Agent 的工具链执行流程
- 检查推理决策是否合理
- 验证异常处理和容错机制

### ITERATION_REFACTOR（迭代重构）
- 运行已有测试确认无功能回归
- 对比重构前后的代码结构改进
- 验证性能或可维护性提升

### PRODUCT_DESIGN（产品设计）
- 重点审查候选人提交的 README/设计文档
- 检查原型/Demo 的完成度和核心场景覆盖
- 评估技术方案可行性和产品思维
